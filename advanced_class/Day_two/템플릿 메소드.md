# 템플릿 메소드(Template Method Pattern) 패턴
- 템플릿 메소드는 알고리즘의 각 단계를 정의하며, 서브클래스에서 일부 단계를 구현할 수 잇도록 유도합니다.

- 알고리즘의 골격을 정의합니다. 템플릿 메소드를 사용하면 알고리즘의 일부 단계를 서브클래스에서 구현할 수 있으며, 알고리즘의 구조는 그대로 유지하면서 알고리즘의 특정 단계를 서브클래스에서 재정의 할 수도 있습니다. 

- 간단하게 말하자면 템플릿 메소드 패턴은 알고맂므의 템플릿(틀)을 만듭니다.

## 템플릿 메소드의 패턴의 장단점

### 장점 
- 하나의 클래스에서 작업을 처리합니다. 알고리즘 독점
- 중독 된 코드가 아닌 서브 클래스에서 코드 재사용
- 알고리즘이 한 군데에 모여 있으므로 유지보수 시에 한부분만 고치면 됨
- 다른 음료도 쉽게 추가할 수 있는 프레임 워크 제공
- 클래스에 알고리즘 지식이 집중되어 일부 구현만 서브 클래스에 의존

### 단점
- 구조가 고정되어 있어 유연성이 떨어질 수 있습니다.
- 알고리즘 단계 변경시 부모 클래스 수정이 필요하다 


## 사용 예시
- 웹 요청 처리(MVC 패턴, 로그인)
- 파일처리/ 데이터 파이프라인
